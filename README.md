# Thermal Human Detection — ADAS

A computer vision system that detects humans from **thermal (infrared) imagery** using YOLOv8.  
Built on the FLIR ADAS multimodal dataset. The model outputs **"Humans detected: N"** or **"No human detected"** for every frame — no other classes are considered.

---

## Why Thermal?

Standard RGB cameras fail in darkness, fog, and glare. Thermal cameras detect **body heat (~37 °C)**, making human detection reliable in conditions where visible-light cameras are blind — exactly the scenarios where ADAS systems need to work most.

---

## Dataset

**FLIR ADAS Thermal Dataset** — dual modality (RGB + Thermal/IR), COCO JSON annotations.

| Split | Modality | Person annotations |
|---|---|---|
| `images_thermal_train` | Thermal | 50,478 |
| `images_thermal_val` | Thermal | 4,470 |
| `video_thermal_test` | Thermal video | test only |

16 object classes in the dataset (car, bike, light, sign, …) — only `person` is used for training.

> **Note:** Raw dataset files (`coco.json`, images) are not tracked by git due to size.  
> Download the FLIR ADAS dataset separately and place it in the project root.

## Quickstart

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

Requires Python 3.10+. A CUDA-enabled GPU is strongly recommended for training.

### 2. Prepare the dataset *(run once)*

```bash
python prepare_data.py
```

Reads `images_thermal_train/coco.json` and `images_thermal_val/coco.json`, filters to the `person` class, converts bounding boxes to YOLO format, and populates `dataset/`.

### 3. Train

```bash
python train.py
```

Common options:

```bash
python train.py --epochs 100 --batch 8 --weights yolov8s.pt
python train.py --resume                          # resume from last checkpoint
python train.py --weights yolov8n.pt --batch 4   # low VRAM setup
```

Best weights saved to `runs/detect/thermal_person/weights/best.pt`.

### 4. Detect humans

```bash
# Folder of images
python detect.py --weights runs/detect/thermal_person/weights/best.pt \
                 --source images_thermal_val/data

# Video file
python detect.py --weights best.pt --source path/to/video.mp4 --save-video

# Live webcam
python detect.py --weights best.pt --source 0

# RTSP stream
python detect.py --weights best.pt --source rtsp://user:pass@ip:port/stream
```

## Model

| Backbone | Pretrained on | Fine-tuned on | Classes |
|---|---|---|---|
| YOLOv8m | ImageNet (RGB) | FLIR thermal frames | 1 (`person`) |

### Choosing a model size

| Flag | Model | Speed | VRAM |
|---|---|---|---|
| `--weights yolov8n.pt` | Nano | ~170 FPS | 4 GB |
| `--weights yolov8s.pt` | Small | ~120 FPS | 4 GB |
| `--weights yolov8m.pt` | Medium *(default)* | ~80 FPS | 6–8 GB |
| `--weights yolov8l.pt` | Large | ~50 FPS | 10 GB |
| `--weights yolov8x.pt` | XLarge | ~30 FPS | 14 GB |

## Output

| Condition | On-screen | Console |
|---|---|---|
| Human(s) present | Green box + **"Humans detected: N"** | `[frame.jpg] --> 2 human(s) detected` |
| No human present | Red banner **"No human detected"** | `[frame.jpg] --> No human detected` |

Training metrics and plots are saved to `runs/detect/thermal_person/`:
- `results.csv` — loss & mAP per epoch
- `confusion_matrix.png`
- `PR_curve.png`, `F1_curve.png`

---

## Key Metrics

| Metric | Meaning | Target |
|---|---|---|
| **mAP50** | Mean Average Precision @ IoU=0.5 | > 0.80 |
| **Precision** | Of all detections, how many are real humans | > 0.85 |
| **Recall** | Of all real humans, how many were found | > 0.80 |

For ADAS safety, **Recall matters more** — missing a pedestrian is worse than a false alarm.

---

## Tuning Tips

- **Too many false positives** → raise `--conf` (e.g. `0.5`)
- **Missing distant/partial humans** → lower `--conf` (e.g. `0.2`)
- **Out of VRAM** → halve `--batch` or switch to a smaller model (`yolov8s`)
- **Wants all 16 classes** → set `TARGET_CLASS_NAME = None` in `prepare_data.py` and update `thermal_person.yaml`
